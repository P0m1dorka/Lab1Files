Это краткий гайд о том, как пользоваться гитом. Я не буду затрагивать тему
создания гит репозитерия и соединения локального репозитория и облочной истории.

Собственно как мы будем работать? 
Я предлагаю самый базовый план, который мы будем использовать после 1 лабы. 
1) Обсудили кт и решили кто что будет делать 
2) Кто-то из нас создаёт базовый репозиторий (пустой проект пайчарма ) 
3) Создатель добавляет остальных в репозиторий через github.com 
4) Я думаю, что использовать консоль будет проще, чем приложение 
(логика приложения GitHub_Desktop очень похожа на консоль). Гитхаб десктоп это 
просто гуи для консоли ахаха
5) Зарегистрироваться в github desktop / через ssh ключ сконектиться с гитом для терминала. Вроде как когда ты регаешься в github desktop, то консольный гит тоже конектится.
6) Вы склонировали репозиторий через команду git clone <адрес репозитория> или
через приложение (File -> Clone repository -> вводите или имя репозитория или в графе url вставляете ссылку). Потом выбирает место где сохранить 
7) Теперь вам нужно создать новую ветку для работы, так как в main мы не будем работать после 1 лабороторной работы.
8) Для того чтобы создать ветку вы нажимаете на плашку "Current branch <имя ветки где вы щас работаете>". У вас открывается мини-меню и вы нажимает new branch и выбираете ветку main. Если во время создания у вас будет выбрать "leave changes on <та ветка где вы щас> или "bring my changes to <ветка, которую вы щас создали>" ". Вы выбираете второй вариант. 
9) Вы создали ветку. Перед тем как выполнить работу лучше нажать на плашку publish branch. 
10) Вы выполнили всю работе или сделали часть работы. 
11) Слева внизу у вас есть надпись Summary (requiered). Вы нажимаете на неё и пишите краткое описание того, что сделали 
12) После этого ещё ниже слева у вас начнёт гореть плашка "Commit to <имя ветки где вы щас работаете>"
13) После этого сверху у вас будет написано что-то в духе push origin или push to <имя вашей ветки>. Что-то подобное. Вы нажимаете на неё 
14) Я сейчас крайне устал писать гайд, но продолжу. 
15) Рано или поздно вы закончите работать. Вы переключаетесь на ветку main. Для этого вы нажимаете на плашу слева сверху "Current branch <имя ветки где вы работаете>" и выбираете ветку main. 
16) Снова нажимаете на плашку Current branch и в самом низу открывшегося меню есть плашка "Choose a branch to merge into main ". Затем выбираете ветку с которой хотите скачать и снизу у вас загорится плашка create merge commit. Нажимаете на неё. 
17) Также слева от create merge commit есть стрелочка вниз, которая позволяет выбрать опции, если хотите, то можете менять их, но и базовая конфигурация, которая выбрана приложением подходит вам. 
18) Дальше вы нажимаете  push origin справа от плашки "Current branch"
